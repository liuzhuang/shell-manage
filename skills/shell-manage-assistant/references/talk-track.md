# ShellManage 产品演示说明

## 30 秒介绍

ShellManage 是一个 macOS 命令工作台。它把项目启动、实时日志、交互终端、内置浏览器和运行监控放在同一个桌面应用中。常用操作通过界面完成，YAML 作为本地配置存储格式。

## 2 分钟演示顺序

1. **导入演示命令**
   - 在「命令」页选择「＋ 添加命令」→「导入演示命令」。
   - 演示条目覆盖后台服务、异常退出和交互终端。
2. **启动并查看日志**
   - 启动 `demo-service`，展示运行状态、PID 和实时日志。
   - 启动 `demo-bad-exit`，展示退出状态。
3. **打开终端和浏览器**
   - 打开 `demo-terminal` 的交互窗口。
   - 对配置了 `webUrl` 的命令使用「内置打开」。
4. **查看监控和设置**
   - 在「监控」页查看本机指标。
   - 在「设置」页切换可视化编辑与 YAML 源码编辑。

## Skill 的作用

`shell-manage-assistant` 可以在已成功加载该 Skill 的运行环境中：

- 解释当前界面和配置字段
- 读取项目文件并生成候选启动命令
- 展示配置差异，在确认后写入
- 根据日志、模式和配置提供排查步骤

公开版本和安装信息只读取 GitHub Releases。网络不可用或无法核验最新正式 Release 时，Skill 只返回发布页面，不猜版本、资产 URL 或校验值。

## 常见问题

### 是否必须手写 YAML？

常用命令、预设和全局设置可以在「设置」页进行可视化编辑。健康检查、终端启动步骤等高级字段仍可能需要使用 YAML 源码编辑。

### 如何开始体验？

「导入演示命令」提供不依赖个人项目路径的示例。完成导入后，可依次查看后台服务、异常退出和交互终端。

### 配置写错后如何处理？

应用保存配置前会执行结构和字段校验。Skill 的配置流程还要求先展示差异、请求确认、写入后复检，并提供回滚方式。

### 哪些 AI 编程客户端可以加载该 Skill？

仓库只提供 Skill 源码和目录示例，不对所有客户端作兼容性保证。是否能够发现和执行 Skill，以实际运行环境的加载规则和验证结果为准。

### ShellManage 从哪里下载？

访问 `https://github.com/liuzhuang/shell-manage/releases`。网络可用时，Skill 可以核验最新的非 draft、非 prerelease Release，并只使用该 Release 返回的真实安装资产；无法在线核验时只提供发布页面。
