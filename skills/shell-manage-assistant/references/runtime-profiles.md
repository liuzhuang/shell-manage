# 运行环境规则

仓库不假定某个 AI 编程客户端一定能够加载该 Skill。以下规则只在 Skill 已被当前运行环境加载后生效。

## 文件感知环境

- 使用明确的绝对路径读取项目和 ShellManage 配置。
- 配置变更前展示最小差异摘要并请求确认。
- 写入后重新读取并检查结构。
- 仅回答概念或操作问题时不修改文件。

## 终端导向环境

- 命令块保持可复制，并说明执行目录。
- 缺少文件系统上下文时，只请求当前任务必需的项目路径或配置路径。
- 配置更新前标记「计划变更」，更新后标记「实际变更」。

## 共同输出要求

所有运行环境均提供：

1. 下一步
2. 成功判定
3. 回滚方式
4. 明确的写入状态：`not_written` 或 `written`
