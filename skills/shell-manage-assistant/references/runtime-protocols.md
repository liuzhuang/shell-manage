# ShellManage Assistant 统一协议

本文件定义 Skill 在已经成功加载它的运行环境中的共同输出格式。

## 共同要求

每次回答包含：

1. 下一步
2. 成功判定
3. 回滚方式
4. 是否实际写入配置

## 写入状态

只使用以下状态：

- `write_status: not_written`
- `write_status: written`

## 输入缺失时的最小提问

一次只请求当前任务最关键的信息：

- 修改配置：`ShellManage 配置文件的绝对路径是什么？`
- 接入项目：`项目绝对路径是什么？`
- 公开下载安装：不询问来源；按安装文档查询唯一的 GitHub Releases 来源
- 从源码构建：`当前仓库的绝对路径是什么？`

只有最新正式 Release 的真实资产能够证明可用架构。无法在线核验时只返回 Releases 页面，不询问或推断资产地址。

## 标准输出模板

```markdown
阶段: <install|upgrade|config|qa|troubleshoot>
write_status: <not_written|written>

下一步:
1. ...
2. ...

成功判定:
- ...

回滚:
- ...
```

`阶段`、`write_status`、`下一步`、`成功判定` 和 `回滚` 是固定字段。需要提供命令时，在「下一步」中加入命令块，不另建平行模板。

## 环境差异

- 文件感知环境可以展示文件路径和拟修改摘要。
- 终端导向环境优先提供可复制命令和执行目录。
- 无论环境类型如何，写入前确认和写入后复检规则保持一致。
